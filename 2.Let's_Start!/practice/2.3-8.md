>$n$ 個の整数の集合 $S$ とある整数 $x$ が与えられた時、$S$ のなかの2個の要素で、それらの和がちょうど $x$ になるものが存在するかどうかを決定するアルゴリズムを記述せよ。<br>
そのアルゴリズムは、最悪実行時間が $\Theta(n \lg n)$ となるようにせよ。

```
TWO-SUM(S, x)
1. ソート S を昇順に（例えばクイックソートなどで）  // O(n lg n)
2. i ← 1、j ← length(S)
3. while i < j:                             // O(n)
4.     sum ← S[i] + S[j]
5.     if sum == x:
6.         return True
7.     else if sum < x:
8.         i ← i + 1
9.     else:
10.        j ← j - 1
11. return False
```

$$
T(n)=O(n \lg n)+O(n)=O(n \lg n)
$$
